# 2.7 Speech Processing: VAD, STT, TTS

For the robot to be operated by voice, spoken audio must be turned into text the agent can process, and the agent's textual reply must be turned back into speech. This is achieved by a pipeline of three stages, shown in Figure 2.17: voice activity detection (VAD) isolates the portions of the microphone stream that actually contain speech, speech-to-text (STT) transcribes that speech into text, and text-to-speech (TTS) synthesizes an audible reply from the agent's answer. Between STT and TTS sits the conversational agent of Section 2.6.

> 🖼️ Figure 2.17 — The voice interaction pipeline: microphone audio → VAD → STT → text → (dialogue agent) → text → TTS → loudspeaker audio (drawn by the group).

Voice Activity Detection. VAD decides, for each short frame of the audio stream, whether it contains human speech or only silence and background noise. It serves two purposes: it prevents non-speech audio from being sent to the (much more expensive) recognizer, and it detects when the speaker has finished a turn. Simple energy-thresholding is fragile in a noisy restaurant, so current practice is to use a small pretrained neural VAD a lightweight network (e.g., of the Silero VAD family [n]) that outputs a speech probability for every frame, which is compared against a decision threshold. Endpointing is then a timing rule: once speech has started, the utterance is considered finished when silence persists longer than a preset duration, with a maximum-length cap so that a single turn cannot run indefinitely.

Speech-to-Text. STT (also called automatic speech recognition, ASR) converts the detected speech segment into a text transcript. Modern recognizers are single end-to-end neural networks, encoder–decoder Transformers built on the same attention mechanism as Section 2.6.1, pretrained on very large multilingual audio–text corpora. Models of the Whisper family [n] are a widely used example and are available through optimized inference runtimes (such as faster-whisper [n]) that run the same pretrained weights with substantially lower latency. In this work such a model is used entirely as-is: no training or fine-tuning is performed; the target language is simply specified (Vietnamese), which improves accuracy over automatic language detection. It must be emphasized that recognition is not error-free background noise, regional accents, and unusual dish names all cause transcription errors. So the downstream components must be designed to tolerate imperfect transcripts, for instance by fuzzy-matching dish names against the menu rather than requiring exact strings.

Text-to-Speech. TTS performs the inverse mapping given the agent's textual reply, it produces a natural-sounding audio waveform.
